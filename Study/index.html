Convolutional Neural Network (CNN)

The operation process of a Convolutional Neural Network (CNN) mainly involves the following steps:

  1. The input layer is responsible for image input.
  2. Then, through the convolution layer, features of the image are extracted. The convolution layer performs convolution operations on the data using a filter/kernel, and through sliding operations, features of the entire image are extracted to generate corresponding feature maps.
  3. Using max pooling or average pooling, downsampling is performed, reducing the size of the feature maps while retaining important features.
  4. The pooled feature maps are passed to a neural network, which predicts the image content.

Recurrent Neural Network (RNN)

The main feature of a Recurrent Neural Network (RNN) is its ability to handle sequential or time-dependent data. Due to its recurrent connections and unfolding, each output depends not only on the current input but also on the hidden state from the previous time step.

However, when dealing with long sequences, due to the unfolding nature, the network may encounter the problem of vanishing/exploding gradients. As shown in the diagram, when there are 50 previous data points, repeated multiplication by (w_2) causes the gradient to shrink to (2^{-50}), leading to the vanishing gradient problem. Similarly, if (w_2) is greater than 1, it leads to exploding gradients.

Long Short-Term Memory (LSTM)

Long Short-Term Memory (LSTM) effectively addresses the vanishing/exploding gradient problem in RNNs. The architecture is shown in the diagram above.

  • The green line at the top represents long-term memory (L), also known as the cell state, which is used to store previously inputted data.
  • The blue section represents the forget gate, which processes the current input through a sigmoid function to generate a corresponding forget rate. This value is multiplied by the long-term memory (L) to indicate how much to “remember.”
    • If the calculation results in 97%, most of the long-term memory will be retained.
    • If the calculation results in 5%, most of the long-term memory will be forgotten.
  • The green and yellow sections represent the input gate. The yellow section computes the potential long-term memory, used to generate the content that the long-term memory (L) needs to remember. The green section computes how much of the potential long-term memory to remember (% Potential Memory To Remember). The two values are multiplied and added to the long-term memory (L).
  • The purple and pink sections represent the output gate. First, it computes how much of the short-term memory to remember (% Potential Memory To Remember), using the input and short-term memory. Then, similar to the long-term memory, this value is used to generate the final prediction.

Word Embedding and Word2Vec

To convert words into corresponding vectors for processing by computers, the Word2Vec method is introduced.

At its core, Word2Vec is still a neural network model, where the initial weights are random. The model is trained via backpropagation, and the final weights, (w), are what we refer to as Word Embeddings. In addition to sequence prediction, there are methods such as Continuous Bag of Words (CBOW), where words are predicted using the surrounding context, and Skip-gram, which predicts the surrounding context given a central word.

In essence, Word2Vec converts words into vector representations that can be understood by computers.

Word2Vec generates low-dimensional, dense vector representations through training, where:

  • Similar words (e.g., “king” and “queen”) are close to each other in vector space.
  • The semantic relationships between words can be expressed via arithmetic operations on vectors (e.g., “king” - “man” + “woman” ≈ “queen”).

Seq2Seq

Seq2Seq is essentially an encoder-decoder architecture. It encodes an input sequence (e.g., using Word2Vec) and feeds it into the encoder part of a neural network for processing. Note that the neural network used here could be RNN, LSTM, or GRU, and it is often multilayered, with the same weights and biases shared across the same layers. The resulting context vector is passed to the decoder, which processes it, feeding into a fully connected layer (another neural network) that finally produces a prediction passed to the next decoder input.

Note: During training, the decoder uses the training token instead of the fully connected layer output as input.

Attention

The major limitation of RNNs is that they rely on a single pathway to handle both long-term memory and short-term memory, leading to the issues of vanishing or exploding gradients. While LSTMs improve this to an extent, they still tend to forget longer-term information when processing large datasets. The Attention mechanism was introduced to address this. Its core idea is to provide a direct pathway between each input and the decoder, allowing the decoder to access the input sequence at every decoding step.

Note: The denominator mainly serves to scale the output to the range (-1, 1), making comparison simpler. In practice, only the numerator (the Dot Product) is often used for comparison.

The computed similarity score is then passed through a SoftMax function, yielding a result that represents the percentage of attention placed on the encoded word when decoding. These weighted scores (for both Let's and go in relation to <EOS>) are combined, yielding an attention value for <EOS>.

Finally, the similarity scores for Let's and go, along with the encoding of <EOS>, are passed into a fully connected layer. After processing, the SoftMax function determines the next predicted word. This output is then used as the new input to the decoder, repeating the process until the final output is produced.

Transformer

To determine the order of input data, positional encoding is introduced. It assigns each embedding a specific positional value using special functions (such as sin(x), cos(x)), allowing the Transformer to maintain the tracking of word order. The positional encoding is shown in the diagram below:

The Self-Attention mechanism is used to examine the similarity between each word in the input sentence (including itself) and the word it represents. Once the similarity between each word and every other word is calculated, this information will be used to determine how the Transformer encodes each word.

For example: The pizza came out of the oven, and it tasted good.

After calculation, the Transformer realizes that it is more similar to pizza rather than oven, meaning that pizza will have a greater influence when the Transformer encodes it.

Simply put, the Query value for the word Let's is first generated, followed by the Key values for both Let's and go. The Query value of Let's is then used to calculate similarity with the Key values of both Let's and go. After calculating the similarity between Let's and every other word, the SoftMax function is applied to compute the “encoding percentage“ of each corresponding word. The Value values of both Let's and go (i.e., all the words) are then multiplied by their respective encoding percentages and summed up. The resulting value is the Self-Attention value of Let's. To summarize the decoder part: The input part of the Transformer consists of word embedding, positional encoding, self-attention, and residual connection, which together enable the Transformer to encode words into numbers.

For the decoder part, the words to be decoded are encoded in the same way as described above. The difference lies in the following:

The Query corresponding to the word input to the decoder must be calculated in conjunction with the Key and Value derived from the encoder, yielding the corresponding Self-Attention value. Similarly, an additional set of Residual Connection values is added, and the result is then passed into a fully connected layer, which ultimately generates the predicted output.

This process is repeated until decoding is complete.

[!IMPORTANT]

In summary, the Transformer uses word embeddings to convert words into numbers, positional encoding to track word order, the self-attention mechanism to monitor word relationships in both input and output sequences, and encoders and decoders to track relationships between input and output phrases, ensuring that no important content is lost. Residual connections allow each sub-unit to focus on a specific part of the problem.
